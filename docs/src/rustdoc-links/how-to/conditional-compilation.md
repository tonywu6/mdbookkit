# Conditional compilation

The preprocessor has some support for generating links to items that are platform- or
feature-specific. You can configure the [features](#specifying-features) to enable when
running `cargo doc`, and build docs for multiple [targets](#specifying-targets) at once.
For more complex scenarios, you can use [multi-stage builds](#multi-stage-builds) to
resolve all links over multiple passes.

## The problem

As mentioned in [Naming items](../naming-items.md#under-the-hood), the preprocessor
resolves links by utilizing compiler artifacts generated by `cargo doc` and `rustdoc`,
where:

> Rustdoc treats your crate the same way the compiler does. Only things from the host
> target are available (or from the given `--target` if present), and everything else is
> “filtered out” from the crate.[^rustdoc-conditional]

Which means that the preprocessor also cannot automatically "see" conditionally-compiled
items without extra configuration. The officially-recommended method is to [add
`#[cfg(doc)]`][cfg-doc] to any such items to enable them during `rustdoc`.

Unfortunately, due to the way the preprocessor works, this is unsupported. This also
will not help if you are trying to mention items from dependencies that are
conditionally-compiled.

In lieu of this, the preprocessor offer several options to control the compilation
process.

## Specifying features

Use `build.features` to enable additionally features:

```toml config-example
[preprocessor.rustdoc-links]
build.features = ["fancy", "clap/unstable-doc"]
# cargo doc --features fancy --features clap/unstable-doc
```

This option corresponds to the [`--feature` option of `cargo doc`][`--features`]. This
example enables the `fancy` feature for your local package and the `unstable-doc`
feature of the `clap` package.

You can also use `build.all-features` and `build.no-default-features`, which correspond
to the [`--all-features`] and [`--no-default-features`] options, respectively.

```toml config-example
[preprocessor.rustdoc-links]
build.all-features = true
```

```toml config-example
[preprocessor.rustdoc-links]
build.no-default-features = true
```
